What is Devfolio? #

Devfolio is an open-source CLI tool that eliminates the friction of building and maintaining a developer portfolio. Instead of manually updating your projects, skills, and statistics, Devfolio automatically syncs with your GitHub profile to create a living, breathing portfolio that stays current.

Built on modern web technologies like Vite, Tailwind CSS, and optionally Framer Motion, Devfolio generates production-ready static sites that are blazing fast, fully customizable, and free to deploy anywhere.

Open Source
Devfolio is completely free and open source under the MIT license. Contributions are welcome!

Key Features #

  • Automatic GitHub Sync: Fetches repositories, stars, languages, and profile data in real-time
  • Multiple Templates: Choose from minimal, modern, or creative portfolio layouts
  • Zero Configuration: Works out of the box with sensible defaults
  • TypeScript Support: Fully typed codebase for better developer experience
  • Static Generation: SEO-friendly static HTML output for optimal performance
  • Dark Mode: Built-in dark mode support across all templates

Quick Start #

Get started with Devfolio in three simple steps. No prior configuration needed—just your GitHub username.

Run the CLI
Open your terminal and run the following command to create a new portfolio:
bash 
$ npx create-devfolio@latest my-portfolio
Answer the prompts
The CLI will ask for your GitHub username and preferred template. Select from available options.
Start developing
Navigate to your project and start the development server:
bash 
cd my-portfolio
npm run dev
Success!
Your portfolio is now running at http://localhost:5173. Open it in your browser to see your live data.

Configuration #

Devfolio uses a simple configuration file devfolio.config.js (or .ts) to customize your portfolio. Here's the complete configuration reference:

Configuration Options #

Option Type Default Description
github.username required string — Your GitHub username
site.title string "Portfolio" Site title
site.description string GitHub bio Meta description
template string "modern" Template name
theme.primaryColor string "#000000" Primary brand color
features.blog boolean false Enable blog section
features.darkMode boolean true Enable dark mode toggle

Example Configuration #

javascript 
export default {
  github: {
    username: 'octocat',
    token: process.env.GITHUB_TOKEN // Optional: for higher API limits
  },
  site: {
    title: 'Alex Developer',
    description: 'Full-stack engineer building tools for the modern web',
    url: 'https://alex.dev'
  },
  template: 'modern',
  theme: {
    primaryColor: '#3b82f6',
    darkMode: true
  },
  features: {
    blog: true,
    analytics: true
  }
}
GitHub API Limits
Unauthenticated requests are limited to 60 per hour. For production builds, we recommend setting a GITHUB_TOKEN environment variable.

CLI Commands #

Devfolio provides several CLI commands to manage your portfolio throughout the development lifecycle.

Command Description
devfolio create [name] Create a new portfolio project
devfolio dev Start development server with hot reload
devfolio build Generate production build
devfolio preview Preview production build locally
devfolio update Update GitHub data and rebuild
devfolio deploy Deploy to configured platform
Pro Tip
Use devfolio update in a CI/CD pipeline to automatically refresh your portfolio data daily.
